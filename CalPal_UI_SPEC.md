# CalPal — UI Design Specification
**版本：** v1.1
**日期：** 2026-03-27
**配合文档：** CalPal_PRD.md / CalPal_SwiftUI_Guide.md

---

## 1. 设计原则

- **沿用 iOS 系统设计语言**：视觉风格、字体、间距、动效完全遵循 Apple Human Interface Guidelines
- **日历优先**：app 首先是一个日历，其次是一个 AI 助理，不是聊天工具
- **语音为主**：核心操作通过长按语音完成，文字输入为辅助
- **不过度设计**：V1 不引入 Liquid Glass、复杂动效等高成本视觉元素，保持系统原生感
- **尽量贴近系统 Calendar**：V1 以苹果自带日历的结构和操作习惯为参考，只做最少量 AI 强化视觉
- **安全区域**：所有可交互元素严格遵守 iOS Safe Area，FAB 按钮距底部 `safeAreaInsets.bottom + 20pt`

---

## 2. 色彩规范

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主题色（按钮、高亮） | `#4A7FD4`（蓝色）| 接近苹果 Calendar 蓝，用户熟悉 |
| 当前时间红线 | `#E24B4A`（红色）| 与系统 Calendar 一致 |
| 录音激活状态 | `#E24B4A`（红色）| FAB 按钮按下变红 |
| 新增事件高亮 | `#4A7FD4` 边框 + `#E6F1FB` 背景 | 预览卡中区分新事件 |
| Apple Calendar 事件 | 跟随用户在系统日历里设置的颜色 | EventKit 提供 `cgColor` |
| Google Calendar 事件 | 跟随 Google Calendar 颜色设置 | API 返回 colorId |
| 背景 | 系统默认（白/深色自适应）| 支持 Dark Mode |
| 文字 | 系统默认文字颜色 | 支持 Dark Mode |

## 3. 核心页面规范

### 3.1 主界面（Main Calendar View）

**布局结构（从上到下）：**

```
┌─────────────────────────┐
│  状态栏（系统）           │
├─────────────────────────┤
│  导航栏                  │
│  [今天]   2026年3月   [≡] │
├─────────────────────────┤
│  紧凑日期 Pager             │
│  ‹ 22 23 24• 25 26 ›       │
├─────────────────────────┤
│                          │
│  Timeline（可上下滚动）   │
│  9 AM │ ██ Team standup  │
│       │                  │
│ 10 AM │                  │
│ ──────┼── 红线（当前时间）│
│ 11 AM │ ██ Lunch         │
│       │                  │
│  ...  │                  │
│                          │
│              [🎤 FAB]    │  ← 悬浮，右下角
└─────────────────────────┘
│  Home Indicator（系统）   │
└─────────────────────────┘
```

**导航栏：**
- 左：「今天」文字按钮（点击跳回当天）
- 中：当前月份标题（`YYYY年M月`）
- 右：菜单图标（进入设置）

**日期 Pager：**
- 采用接近系统 Calendar 的紧凑横向日期结构，不再使用大面积日期卡片
- 中间展示当前选中的 5 天窗口，左右两端轻微露出前后日期，暗示可横向翻页
- 当前选中日期：蓝色圆形背景高亮
- 相邻日期：仅数字 + 极轻的透明度变化
- 有事件的日期可用小圆点提示，不额外增加信息负担
- 支持左右滑动切换日期范围；交互优先做成系统日历那种自然分页，不做明显 carousel 动效
- 视觉原则：弱化容器、弱化阴影、弱化边框，让焦点回到 Timeline 本身

**Timeline：**
- 左侧时间标签：每小时一行，`8px` 灰色文字
- 每小时高度：`60pt`（与 iOS Calendar 保持一致）
- 事件色块：圆角 `4pt`，使用日历源颜色，内显示事件标题 + 时间
- 当前时间红线：`1.5pt` 红色横线 + 左侧红色圆点
- 支持双指捏合缩放时间密度（V2 功能，V1 固定高度）
- 主 Timeline 允许左右切天，但视觉上不额外叠加复杂分页组件；应让用户感觉自己仍在系统式日历里

**FAB 麦克风按钮：**
- 尺寸：`56pt × 56pt`，圆形
- 位置：右下角，距右边 `20pt`，距底部 `safeAreaInsets.bottom + 20pt`
- 颜色：蓝色 `#4A7FD4`，白色麦克风图标
- 阴影：轻微（遵循 iOS Material 规范）
- 旁边辅助按钮：「Type」文字按钮（小型，灰色描边），距 FAB 左侧 `16pt`

### 3.2 语音输入状态（PTT Recording State）

**触发：长按 FAB 按钮**

**状态变化：**
- FAB 按钮：蓝色 → 红色
- 按钮内图标：麦克风 → 波形动画（7根竖条，高度随音量变化）
- 按钮外：两层半透明涟漪圆环扩散动画（表示「正在监听」）
- FAB 左上方：红色标签 `● 松手结束`
- Timeline 区域：正常显示，**可继续上下滚动**（这是关键体验）
- 日期 Pager 保持可见，录音态下仍保留“当前所处日期上下文”

**释放触发：**
- 停止录音 → 语音转文字 → 传入 Foundation Models 解析
- 按钮状态恢复蓝色
- 显示加载指示（按钮内转圈）

**取消手势：**
- 长按后向上滑动超过 `60pt` → 取消录音（参考微信语音消息交互）
- 显示提示：「松手取消」

### 3.3 确认底部卡片（Confirmation Sheet）

**触发：** AI 解析出需要确认的操作（冲突 / 周期事件 / 删除操作）

**展示方式：** `sheet()` 底部弹出，可下拉关闭

**卡片内容（添加场景）：**

```
┌─────────────────────────────┐
│  ▬▬▬  （拖动指示条）         │

│  明天 · 下午 2:30–3:30       │  ← 时间（可点击编辑）
│  Apple Calendar             │  ← 目标日历（可切换）
│                             │

│  [修改时间]  [取消]          │  ← 次级按钮，并排
```

### 3.4 日程对话分屏视图（Unified Split View）

**触发：**
- AI 处理结果后，需要在日历上直观展示变更
- 对话式规划建议
- 对话式复盘后补排建议

**布局：上 55% 日历预览 / 下 45% 对话记录**

```
┌─────────────────────────────┐
│  状态栏 + 导航栏              │
├─────────────────────────────┤
│  ── 规划建议 / 复盘建议 ──   │  ← 上方区域（55%）
│                             │
│  Timeline（聚焦变更时间段）   │
│  现有事件（正常显示）         │
│  ✦ 新增 / 建议事件高亮       │
│                             │
│  [确认] [修改时间] [取消]    │  ← 内嵌确认卡（精简版）
├═════════════════════════════╡  ← 分割线（蓝色 2pt）
│  ── 对话 ──（灰色标签）       │  ← 下方区域（45%）
│                             │
│  AI：给出建议                 │  ← 左对齐气泡（灰色）
│  用户：确认 / 改时间 / 跳过   │  ← 右对齐气泡（蓝色）
│                             │
│                    [🎤 FAB] │  ← FAB 缩小版（40pt）
└─────────────────────────────┘
│  Home Indicator              │
└─────────────────────────────┘
```

**用途定义：**
- 这是 V1 唯一需要重点设计的“对话页”形态
- 所有规划、补排、复盘后建议，统一收敛为同一套 split view 模板
- 只允许文案、预览时间段、按钮标签随场景变化，不再单独设计 Morning / Evening 独立页面

**分屏关闭：**
- 用户确认/取消操作后
- 向下滑动对话区域
- 点击导航栏返回按钮

### 3.5 统一对话页变体（Conversation Variants）

V1 不再单独设计“早间规划页”和“晚间复盘页”，统一用 `Unified Split View` 承载两类场景：

**变体 A：规划建议**
- 顶部标签：`规划建议`
- 预览区域聚焦“今天下午”或“接下来可安排时段”
- 典型按钮：`采纳` / `改时间` / `略过`
- 对话语气偏主动建议，如“这段时间适合安排深度工作或健身”

**变体 B：复盘后补排**
- 顶部标签：`复盘建议`
- 预览区域聚焦“明天早上”或“下一次补做机会”
- 典型按钮：`采纳` / `稍后` / `忽略`
- 对话语气偏回顾后补偿，如“今天未完成的事项建议明早先锁定”

## 4. 交互规范

### 4.1 手势

| 手势 | 区域 | 效果 |
|------|------|------|
| 长按（> 0.3 秒） | FAB 按钮 | 开始语音录音 |
| 松手 | FAB 按钮 | 停止录音，提交 |
| 上滑（> 60pt）后松手 | FAB 按钮（长按中） | 取消录音 |
| 上下滑动 | Timeline | 查看不同时段（录音时仍可滑动）|
| 左右滑动 | 日期 Pager / Timeline | 切换前后日期 |
| 下拉 | 确认卡片 | 关闭卡片（等同取消）|
| 点击事件色块 | Timeline | 查看事件详情（系统 EKEventViewController）|

**日期切换交互决策：**
- 以系统 Calendar 的操作感为准，默认允许用户在顶部日期 Pager 左右滑动切天
- 如果主 Timeline 实现横向分页，也应与顶部日期同步，而不是出现两套独立状态
- 不为切天额外设计强视觉教育组件，避免让主页看起来像 AI 仪表盘而不是日历

### 4.2 反馈

| 操作 | 触觉反馈 | 视觉反馈 |
|------|---------|---------|
| 长按 FAB 触发录音 | `UIImpactFeedbackGenerator(.medium)` | 按钮变红 + 涟漪 |
| 松手停止录音 | `UIImpactFeedbackGenerator(.light)` | 按钮恢复蓝色 |
| 事件写入成功 | `UINotificationFeedbackGenerator(.success)` | Toast：「已添加 ✓」|
| 事件删除成功 | `UINotificationFeedbackGenerator(.warning)` | Toast：「已删除」|
| 解析失败 | `UINotificationFeedbackGenerator(.error)` | Toast：「未能理解，请重试」|

### 4.3 Toast 通知（轻量提示）

- 位置：屏幕顶部（导航栏下方）
- 持续：2 秒后自动消失
- 不阻断用户操作
- 样式：圆角胶囊，系统半透明材质（`UIBlurEffect`）

## 5. 组件规范

### 5.1 FAB 按钮状态

```swift
// 三个状态
enum FABState {
    case idle          // 蓝色，麦克风图标
    case recording     // 红色，波形动画，涟漪圆环
    case processing    // 蓝色，转圈加载指示
}
```

### 5.2 事件色块

- 高度：`max(事件时长 × 每分钟px, 30pt)`（最低 30pt，防止太小点不到）
- 超短事件（< 30 分钟）：只显示标题，不显示时间
- 超长标题：截断 + `...`
- 跨天事件：在每天开头显示继续标记

### 5.3 确认卡片高度

- 最小高度：`240pt`
- 内容自适应

## 6. 当前行动指南

### 6.1 视觉优先级
- 优先把主页做得更像系统日历，而不是加更多 AI 装饰
- AI 的存在感集中在 split view 和确认卡，不放大到主页信息结构里
- 日期 Pager、Timeline、FAB 是 V1 三个最重要的界面锚点

### 6.2 组件收敛策略
- `Main Calendar View` 作为主模板
- `PTT Recording State` 只是在主模板上替换 FAB 与提示层
- `Unified Split View` 作为唯一对话页模板
- Morning / Evening 仅保留为内容场景名，不保留为独立页面名

### 6.3 设计到开发的映射原则
- 每个视觉组件都应能对应到一个独立 SwiftUI 组件
- 日期 Pager 与 Timeline 必须共享同一日期状态源
- 对话页中的预览区与主页 Timeline 应共享同一套事件块视觉规则
- 不超过屏幕的 60%

## 6. 深色模式（Dark Mode）

完全跟随系统，使用语义化颜色：
- 所有背景色使用 `UIColor.systemBackground` / `.secondarySystemBackground`
- 文字使用 `UIColor.label` / `.secondaryLabel`
- 分割线使用 `UIColor.separator`
- 主题蓝 `#4A7FD4` 在深色模式下保持不变（同 iOS Calendar）

---

## 7. 与 iOS 系统 Calendar 的视觉对齐

V1 目标是让用户感觉 CalPal 是系统 Calendar 的超级版本，而不是一个陌生的第三方 app。

| 元素 | 对齐方式 |
|------|---------|
| 字体 | SF Pro（系统默认）|
| 时间格式 | 跟随系统设置（12h / 24h）|
| 日期格式 | 跟随系统语言（中文 / 英文）|
| 日历颜色 | 直接使用 EventKit 提供的 `cgColor` |
| 事件详情界面 | 直接使用 `EKEventViewController`（系统原生）|
| 权限弹窗 | 使用 iOS 系统弹窗，不自定义 |
